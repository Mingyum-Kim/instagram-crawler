이 프로젝트를 아래 맥락으로 이어서 진행해줘.

[제품 목표]
인스타그램 저장 게시물/릴스/DM 정보를 자동 수집·요약·분류해 웹에서 재탐색 가능하게 하는 개인용 서비스.

[확정 요구사항]
- 웹 우선
- 일반 사용자 대상
- 검색 기능 없음
- 저장 게시물/릴스/정보성 DM 텍스트 전량 스캔
- 자동 카테고리 분류
- 주간 웹 알림
- 온보딩 3단계 후 동기화 시작
- 동기화 전 필수 권한 동의

[기술 스택]
- Frontend: Next.js
- Backend: FastAPI
- DB: PostgreSQL
- Queue/Cache: Redis
- Auth: JWT + httpOnly cookie

[현재 상태]
- PRD/IA/DB/API/컴포넌트 구조 초안 정리됨
- 구현 템플릿(온보딩/대시보드/인증) 초안 존재
- 실연동/실데이터/워커/스케줄러는 완성 필요

[이번 세션 목표]
1) docker-compose + make up 자동 실행 완성
2) sync/analysis/digest worker 연결
3) 대시보드/카테고리 실데이터 연동
4) 개인정보(연동해제/삭제요청) 플로우 완성
5) 베타 배포 전 점검 체크리스트 통과

가능하면 변경사항은 파일 단위로 제시하고, 실행 커맨드와 검증 방법까지 같이 제시해줘.
